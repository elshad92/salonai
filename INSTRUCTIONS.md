## 1. CLAUDE.md — Правила проекта

### Основная цель
SaaS платформа с AI сотрудниками для салонов красоты. Без комиссий. Рынок — США и Европа.

### Технологический стек
- **Frontend**: Vite + React, хост на Netlify
- **Backend & DB**: Supabase (auth + database)
- **Стили**: Apple/Tesla минимализм (белый фон, чёрный текст, акцент #C8A96E)
- **Язык интерфейса**: английский

### Структура проекта
- `src/pages/Landing.jsx` – лендинг
- `src/pages/Login.jsx` – Google auth через Supabase
- `src/pages/Dashboard.jsx` – дашборд владельца (реальные данные из Supabase)
- `src/pages/Booking.jsx` – форма записи клиента
- `src/lib/supabase.js` – клиент Supabase

### Правила для Claude
- **Язык общения: русский** — ВСЕГДА пиши процесс работы, объяснения, планы и ответы на русском языке. Код и технические термины оставляй на английском.
- Всегда проверяй, что код не содержит секретов (API keys, пароли)
- Никогда не предлагай использовать `rm -rf` без явного подтверждения
- Для правок в React-компонентах – используй функциональные компоненты и хуки
- Для стилей – Tailwind CSS (если используется) или чистый CSS с переменными
- Все новые фичи сначала обсуждай в плане (Plan → Implement → Verify)
- Перед отправкой PR – запусти линтер и проверь типы (TypeScript)

**Следующие шаги:**
1. Подключить DeepSeek или GPT-5.5 к Cursor
2. Сделать AI Администратора (WhatsApp бот)
3. Product Hunt запуск

### decisions.md — Архитектурные решения
- **Почему Supabase?** – Бесплатный старт, встроенная аутентификация, realtime, меньше кода.
- **Почему Netlify?** – Простота деплоя, бесплатный HTTPS, интеграция с GitHub.
- **Почему SalonAI?** – Альтернатива без комиссий, AI сотрудники как USP.
- **Как управлять выгоранием?** – Одна задача за раз, маленькие шаги, видимый результат каждую сессию.

### insights.md — Находки и грабли
- Не хранить секреты в .env на клиенте – использовать Supabase RLS.
- При обновлении Dashboard всегда проверять права доступа (владелец видит только свои салоны).
- Google Auth требует точного совпадения redirect URI – добавить Netlify URL в консоль.
- При работе с Cursor + DeepSeek через прокси – обязательно отключать другие модели, иначе ошибка.
- Не тратить время на "идеальный код" на старте – сначала работающий MVP.

## 4. Слэш-команды (15 шт)

### /ghost — Без объяснений
Отвечай максимально кратко. Без пояснений, без рассуждений, без лишних слов. Только сухой результат – код, команда, исправление, факт.

### /viral — Вирусный контент
Перепиши текст/идею так, чтобы он стал максимально вирусным. Используй хуки, эмоциональные триггеры, эффект неожиданности, срочность, социальное доказательство. Длина: не более 200 слов.

### /hook — Сильные первые строки
Сгенерируй 5 вариантов сильных первых фраз (хуков) для контента. Каждый хук должен вызывать любопытство, возмущение или желание узнать больше.

### /stepbystep — Пошаговое решение
Разбей решение задачи на последовательные шаги. Каждый шаг должен быть конкретным, выполнимым и пронумерованным.

### /optimizer — Оптимизация
Оптимизируй текст/идею/процесс. Улучши чёткость, структуру, лаконичность, убери лишнее, усиль аргументацию. Покажи исходный и оптимизированный вариант.

### /eli5 — Простыми словами
Объясни сложную тему так, будто мне 5 лет. Используй аналогии, избегай жаргона, будь наглядным.

### /deepdive — Глубокий разбор
Сделай детальный, экспертный разбор темы. Включи: первопричины, нюансы, крайние случаи, разные точки зрения, ссылки на источники.

### /monetize — Монетизация
Для идеи/продукта предложи 5 конкретных способов монетизации. Для каждого укажи модель оплаты, аудиторию и примерный доход.

### /startup — Идеи стартапов
Сгенерируй 3 идеи стартапов на основе темы/проблемы. Для каждой: проблема, решение, ЦА, бизнес-модель, MVP.

### /debug — Поиск ошибок
Проанализируй код/конфигурацию/логику. Найди все потенциальные ошибки. Для каждой объясни причину и предложи исправление.

### /sales — Продающие тексты
Напиши продающий текст (оффер, питч, пост). Структура: боль → решение → выгоды → призыв к действию.

### /contrarian — Критика
Выступи в роли критика. Найди слабые места, недостатки, скрытые риски и контраргументы. Цель – показать, что может пойти не так.

### /framework — Структурирование
Возьми хаотичный набор мыслей/данных/идей и структурируй их в виде чёткой системы или фреймворка.

### /checklist — Чек-лист
Преврати задачу/процесс в готовый чек-лист. Каждый пункт — конкретное действие, которое можно проверить.

### /firstprinciples — Первые принципы
Разбери проблему/идею с помощью «первых принципов»:
1. Какие базовые факты абсолютно истинны?
2. Какие допущения можно отбросить?
3. Какие новые решения появляются?
